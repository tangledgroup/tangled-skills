# Full-Text Search with FTS5

Comprehensive guide to SQLite's FTS5 full-text search engine, including table creation, tokenizers, query syntax, ranking, configuration, and advanced features.

## Overview

FTS5 is SQLite's advanced full-text search virtual table module, providing:

- **Fast text searching** across large datasets
- **Ranking algorithms** to order results by relevance
- **Prefix and wildcard searches**
- **Phrase matching** with proximity operators
- **Custom tokenizers** for different languages
- **Snippet generation** showing match context
- **Highlighting** of matched terms
- **Contentless tables** for metadata-only storage

## Creating FTS5 Tables

### Basic Table Creation

```sql
-- Simple single-column FTS5 table
CREATE VIRTUAL TABLE documents USING fts5(
    content
);

-- Multi-column table
CREATE VIRTUAL TABLE articles USING fts5(
    title,
    body,
    tags
);

-- Insert data
INSERT INTO articles VALUES (
    'SQLite Full-Text Search Guide',
    'FTS5 provides powerful full-text search capabilities in SQLite',
    'database search sqlite'
);

INSERT INTO articles VALUES (
    'JSON Processing in SQLite',
    'SQLite offers comprehensive JSON support with json() functions',
    'json data processing'
);
```

### Table Options

```sql
-- Specify tokenizer
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    tokenize='unicode61'
);

-- Enable prefix matching
CREATE VIRTUAL TABLE products USING fts5(
    name,
    description,
    prefix=3  -- Enable prefix search for terms >= 3 chars
);

-- Contentless table (metadata only)
CREATE VIRTUAL TABLE docs_index USING fts5(
    title,
    content,
    content=''  -- No external content table
);

-- With external content table
CREATE VIRTUAL TABLE docs USING fts5(
    title,
    body,
    content='docs_table',      -- External content
    content_rowid='rowid'      -- Rowid column name
);

-- Configure language id
CREATE VIRTUAL TABLE articles USING fts5(
    content,
    content='articles_table',
    langid='lang_id'           -- Column for language ID
);
```

### Column Options

```sql
-- UNINDEXED column (stored but not searched)
CREATE VIRTUAL TABLE docs USING fts5(
    title,
    author UNINDEXED,  -- Store but don't index
    content
);

-- NOTINDEX is alias for UNINDEXED
CREATE VIRTUAL TABLE products USING fts5(
    name,
    sku NOTINDEX,
    description
);
```

## Tokenizers

### Built-in Tokenizers

#### unicode61 (Default)

Unicode-aware tokenizer with language support:

```sql
-- Default tokenizer
CREATE VIRTUAL TABLE docs USING fts5(content);

-- Explicit unicode61
CREATE VIRTUAL TABLE articles USING fts5(
    content,
    tokenize='unicode61'
);

-- With language (e.g., English stemming)
CREATE VIRTUAL TABLE english_docs USING fts5(
    content,
    tokenize='unicode61 "en"'
);

-- With specific options
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    tokenize='unicode61 "separators=._ remove_diacritics=1"'
);
```

**Unicode61 Options:**
- `language` - Language code for stemming (e.g., 'en', 'de', 'fr')
- `separators` - Custom separator characters
- `remove_diacritics` - Strip diacritical marks (0 or 1)
- `tokenchars` - Additional token characters
- `sepchars` - Additional separator characters

#### unicode51

Legacy Unicode tokenizer (no stemming):

```sql
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    tokenize='unicode51'
);
```

#### ascii

Simple ASCII tokenizer:

```sql
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    tokenize='ascii'
);
```

#### porter

Stemmer for English (legacy):

```sql
CREATE VIRTUAL TABLE english_docs USING fts5(
    content,
    tokenize='porter'
);
```

### Custom Tokenizers

```c
// C code to implement custom tokenizer
typedef struct Fts5Tokenizer Fts5Tokenizer;
typedef struct Fts5Token Fts5Token;

struct Fts5Token {
    int nToken;           // Token length in bytes
    const unsigned char *pToken;  // Token data
    int nStart;           // Start offset in original text
};

struct Fts5Tokenizer {
    void *pUserData;
    int (*xDelete)(void *);
    int (*xTokenize)(void *, const char *p, int n, void *pCtx, 
                     int (*xemit)(void *, int, const char*, int));
};

// Register custom tokenizer
sqlite3_create_vtable(db, "my_tokenizer", &tokenizer_module, NULL);
```

### Tokenizer Comparison

| Tokenizer | Language Support | Stemming | Speed | Use Case |
|-----------|-----------------|----------|-------|----------|
| unicode61 | Yes (50+ langs) | Yes | Fast | Most applications |
| unicode51 | No | No | Fastest | Simple ASCII text |
| ascii | No | No | Fastest | ASCII-only content |
| porter | English only | Yes | Medium | English documents |

## Search Queries

### Basic Search

```sql
-- Single term search
SELECT * FROM articles WHERE articles MATCH 'database';

-- Multiple terms (AND)
SELECT * FROM articles WHERE articles MATCH 'SQLite database';

-- Exact phrase (double quotes)
SELECT * FROM articles WHERE articles MATCH '"full text search"';

-- Column-specific search
SELECT * FROM articles WHERE title MATCH 'SQLite';
SELECT * FROM articles WHERE body MATCH 'database engine';
```

### Boolean Operators

#### OR Operator

```sql
-- OR between terms
SELECT * FROM articles WHERE articles MATCH 'SQLite OR PostgreSQL';

-- Complex OR expressions
SELECT * FROM articles WHERE articles MATCH '(SQLite OR PostgreSQL) AND database';
```

#### NOT Operator

```sql
-- Exclude term
SELECT * FROM articles WHERE articles MATCH 'database NOT SQLite';

-- Multiple exclusions
SELECT * FROM articles WHERE articles MATCH 'search NOT (MySQL OR PostgreSQL)';
```

### Prefix Searches

```sql
-- Prefix search (asterisk)
SELECT * FROM products WHERE products MATCH 'data*';
-- Matches: data, database, dataset, datatypes

-- Multi-term prefix
SELECT * FROM articles WHERE articles MATCH 'full* text*';

-- Column-specific prefix
SELECT * FROM products WHERE name MATCH 'widget*';

-- Note: Requires prefix= option in CREATE VIRTUAL TABLE
CREATE VIRTUAL TABLE products USING fts5(name, prefix=3);
```

### Wildcard Searches

```sql
-- Any character wildcard (question mark)
SELECT * FROM products WHERE products MATCH 'w?dget';
-- Matches: widget, wadget, wodget

-- Combined wildcards
SELECT * FROM docs WHERE docs MATCH 't?st*d?t';
```

### Proximity Operators

#### NEAR Operator

```sql
-- Terms within default distance (usually 8 positions)
SELECT * FROM articles WHERE articles MATCH 'SQLite NEAR database';

-- Specific distance
SELECT * FROM articles WHERE articles MATCH 'SQLite NEAR(5) database';
-- Matches if within 5 tokens of each other

-- Multiple terms in proximity
SELECT * FROM articles WHERE articles MATCH 'full NEAR(3) text NEAR(3) search';

-- Column-specific proximity
SELECT * FROM articles WHERE body MATCH 'performance NEAR(2) optimization';
```

### Boolean Combinations

```sql
-- Complex query with AND, OR, NOT
SELECT * FROM articles 
WHERE articles MATCH '(SQLite OR PostgreSQL) AND database NOT tutorial';

-- With phrases and proximity
SELECT * FROM articles 
WHERE articles MATCH '"full text" NEAR search AND (SQLite OR MariaDB)';

-- Grouped expressions
SELECT * FROM docs 
WHERE docs MATCH '(fast NEAR performance) OR ("query optimization")';
```

## Ranking Results

### Default Ranking

FTS5 uses BM25 algorithm by default:

```sql
-- Results ordered by relevance (most relevant first)
SELECT * FROM articles 
WHERE articles MATCH 'database optimization'
ORDER BY rank;

-- With limit
SELECT * FROM articles 
WHERE articles MATCH 'SQLite tutorial'
ORDER BY rank 
LIMIT 10;
```

### Custom Ranking Functions

```sql
-- BM25 ranking (default)
SELECT rank, title, snippet(articles) 
FROM articles 
WHERE articles MATCH 'database'
ORDER BY rank;

-- Alternative: order by relevance score
SELECT * FROM articles 
WHERE articles MATCH 'search'
ORDER BY rank ASC;  -- Lower rank = more relevant
```

### Ranking Options

FTS5 ranking can be configured:

```sql
-- Create table with custom ranking
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    rank='bm25(docs, 10.0, 5.0, 0.3)'  -- k=10, b=0.5, d=0.3
);

-- Common BM25 parameters:
-- k (default 1.2): Term frequency saturation
-- b (default 0.75): Length normalization
-- d (default 0.5): Query length normalization
```

## Snippets and Highlighting

### Generating Snippets

```sql
-- Basic snippet
SELECT 
    title,
    snippet(articles) AS excerpt
FROM articles 
WHERE articles MATCH 'database';

-- Example output:
-- "...provides powerful full-text search capabilities in SQLite..."

-- With custom tokens before/after
SELECT 
    title,
    snippet(articles, '[...]', '...', 5) AS excerpt
FROM articles 
WHERE articles MATCH 'optimization';

-- Parameters:
-- snippet(table, start, end, max_token_count)
```

### Highlighting Matches

```sql
-- Highlight matched terms with <b> tags
SELECT 
    title,
    snippet(articles, '<b>', '</b>') AS highlighted
FROM articles 
WHERE articles MATCH 'SQLite';

-- Example: "...<b>SQLite</b> provides powerful..."

-- Custom highlight markers
SELECT 
    snippet(body, '[HIGHLIGHT]', '[/HIGHLIGHT]') AS excerpt
FROM documents 
WHERE documents MATCH 'search';
```

### Snippet Options

```sql
-- Default snippet (30 chars before and after)
snippet(articles);

-- More context (50 chars)
snippet(articles, '...', '...', 10);

-- No ellipsis
snippet(articles, '', '', 5);

-- Multiple snippets per row (for multiple matches)
SELECT 
    title,
    group_concat(snippet(body, '<mark>', '</mark>'), ' | ') AS excerpts
FROM documents 
WHERE documents MATCH 'database'
GROUP BY title;
```

### boundaries() Function

Finds word boundaries for custom snippet generation:

```sql
-- Find boundary positions
SELECT 
    title,
    boundaries(articles, 'MATCH_PATTERN') AS positions
FROM articles;

-- Use with substr for custom extraction
SELECT 
    title,
    substr(
        content,
        boundaries(articles, 'pattern', 1),
        100
    ) AS custom_snippet
FROM articles 
WHERE articles MATCH 'pattern';
```

## Query Statistics

### bm25() Function

Calculate BM25 score for documents:

```sql
-- Calculate relevance score
SELECT 
    id,
    title,
    bm25(articles) AS score
FROM articles 
WHERE articles MATCH 'database optimization'
ORDER BY score
LIMIT 10;

-- Lower score = more relevant
```

### offset() Function

Get term positions in document:

```sql
-- Find where terms appear
SELECT 
    id,
    title,
    offset(articles) AS term_positions
FROM articles 
WHERE articles MATCH 'SQLite';

-- Returns: term,table,column,rowid,start,end
```

## Boolean Expressions

### Expression Syntax

FTS5 supports complex boolean expressions:

```sql
-- AND (default, space-separated)
SELECT * FROM docs WHERE docs MATCH 'fast efficient database';

-- OR (explicit)
SELECT * FROM docs WHERE docs MATCH 'SQLite OR PostgreSQL OR MySQL';

-- NOT (exclusion)
SELECT * FROM docs WHERE docs MATCH 'database NOT tutorial';

-- Phrases (quoted)
SELECT * FROM docs WHERE docs MATCH '"relational database"';

-- Prefix terms
SELECT * FROM products WHERE products MATCH 'data* proces*';

-- Proximity
SELECT * FROM docs WHERE docs MATCH 'performance NEAR(3) optimization';

-- Grouped expressions
SELECT * FROM docs 
WHERE docs MATCH '(SQLite OR PostgreSQL) AND "open source"';

-- Mixed operators
SELECT * FROM docs 
WHERE docs MATCH '(fast NEAR efficient) AND NOT (commercial OR proprietary)';
```

### Precedence

Operator precedence (highest to lowest):
1. Phrases (`"term1 term2"`)
2. NOT
3. AND
4. NEAR
5. OR

## Contentless Tables

### Creating Contentless Tables

Store all data within FTS5:

```sql
-- Contentless table
CREATE VIRTUAL TABLE snippets USING fts5(
    title,
    content,
    content=''  -- Empty string means contentless
);

-- Insert data (all stored in FTS5)
INSERT INTO snippets VALUES ('Article Title', 'Full article content here');

-- Query normally
SELECT * FROM snippets WHERE snippets MATCH 'article';
```

### Contentless with Metadata

```sql
-- Store additional metadata in FTS5
CREATE VIRTUAL TABLE docs USING fts5(
    title,
    body,
    author UNINDEXED,  -- Stored but not indexed
    date UNINDEXED,    -- Stored but not indexed
    content=''
);

INSERT INTO docs VALUES (
    'Database Guide',
    'Complete guide to databases...',
    'John Doe',
    '2024-01-15'
);

-- Access metadata
SELECT title, author, date FROM docs WHERE docs MATCH 'database';
```

### External Content Tables

Reference external tables for content:

```sql
-- Create external content table
CREATE TABLE articles_table (
    rowid INTEGER PRIMARY KEY,
    title TEXT,
    body TEXT,
    author TEXT,
    published_date TEXT
);

-- Create FTS5 index referencing external table
CREATE VIRTUAL TABLE articles USING fts5(
    title,
    body,
    content='articles_table',
    content_rowid='rowid'
);

-- Insert into external table (FTS5 indexes automatically)
INSERT INTO articles_table VALUES (
    1, 'SQLite Tutorial', 'Learn SQLite...', 'Alice', '2024-01-01'
);

-- Or insert directly into FTS5 table
INSERT INTO articles VALUES (
    'PostgreSQL Guide',
    'Comprehensive PostgreSQL tutorial...',
    'Bob',
    '2024-01-15'
);
```

## Language IDs

### Multi-Language Support

Store language ID for each document:

```sql
-- Create table with language ID column
CREATE VIRTUAL TABLE multilingual_docs USING fts5(
    content,
    langid=lang_id  -- Specify language ID column name
);

-- Insert with language ID (0 = default/unspecified)
INSERT INTO multilingual_docs VALUES ('Hello world', 0);           -- Default
INSERT INTO multilingual_docs VALUES ('Bonjour le monde', 1);      -- Language 1
INSERT INTO multilingual_docs VALUES ('Hola mundo', 2);            -- Language 2

-- Search within specific language
SELECT * FROM multilingual_docs 
WHERE multilingual_docs MATCH 'hola'
AND langid = 2;
```

## Advanced Features

### Auxiliary Functions

Custom functions for ranking and filtering:

```c
// C code example for auxiliary function
typedef struct Fts5Auxdata Fts5Auxdata;

struct Fts5Auxdata {
    int (*xCompare)(void *, void *, void *);
    void (*xDelete)(void *);
};

// Register auxiliary function
sqlite3_create_function(db, "my_rank", 0, SQLITE_UTF8,

```

### Using Auxiliary Functions

```sql
-- Apply custom ranking
SELECT * FROM articles 
WHERE articles MATCH 'database'
ORDER BY my_rank();

-- Filter by auxiliary function
SELECT * FROM docs 
WHERE docs MATCH 'tutorial'
AND date_filter('2024-01-01', '2024-12-31');
```

### Phrase Search with Column Names

```sql
-- Phrase across multiple columns
CREATE VIRTUAL TABLE articles USING fts5(title, body);

-- Search phrase in title only
SELECT * FROM articles WHERE title MATCH '"exact phrase"';

-- Search in specific column
SELECT * FROM articles WHERE body MATCH 'full text search';

-- Phrase spanning columns (if terms appear in order)
SELECT * FROM articles WHERE articles MATCH '"title body"';
```

### Term Lists

```sql
-- Get list of all terms in index
SELECT * FROM docs WHERE docs MATCH 'term:*';

-- Count documents containing term
SELECT count(*) FROM docs WHERE docs MATCH 'database';

-- Find most common terms (requires additional processing)
SELECT 
    value AS term,
    count(*) AS doc_count
FROM docs, json_each('["term1", "term2"]')
WHERE docs MATCH value
GROUP BY value
ORDER BY doc_count DESC;
```

## Performance Tips

### Indexing Strategy

```sql
-- Create indexes for frequently searched terms
CREATE VIRTUAL TABLE products USING fts5(
    name,
    description,
    prefix=4  -- Enable prefix search for common patterns
);

-- Use UNINDEXED for stored but not searched data
CREATE VIRTUAL TABLE articles USING fts5(
    title,
    content,
    author UNINDEXED,
    published_date UNINDEXED
);
```

### Batch Operations

```sql
-- Insert in transactions for better performance
BEGIN TRANSACTION;

INSERT INTO articles VALUES ('Title 1', 'Content 1...');
INSERT INTO articles VALUES ('Title 2', 'Content 2...');
INSERT INTO articles VALUES ('Title 3', 'Content 3...');

COMMIT;

-- Bulk delete in transaction
BEGIN;
DELETE FROM articles WHERE articles MATCH 'obsolete';
DELETE FROM articles WHERE articles MATCH 'deprecated';
COMMIT;
```

### Query Optimization

```sql
-- Use specific column names when possible
SELECT * FROM articles WHERE title MATCH 'SQLite';  -- Faster
-- Instead of:
SELECT * FROM articles WHERE articles MATCH 'SQLite';  -- Searches all columns

-- Limit results early
SELECT * FROM articles 
WHERE articles MATCH 'database'
ORDER BY rank 
LIMIT 20;

-- Use prefix search sparingly (slower)
CREATE VIRTUAL TABLE docs USING fts5(content, prefix=0);  -- Disable if not needed
```

## Troubleshooting

### Common Issues

**Search returns no results:**
```sql
-- Check if data was inserted
SELECT count(*) FROM articles;

-- Verify term exists in index
SELECT * FROM articles WHERE articles MATCH 'term:*';

-- Try different tokenizer
CREATE VIRTUAL TABLE test_docs USING fts5(
    content,
    tokenize='unicode51'  -- Simpler tokenizer for testing
);
```

**Poor ranking results:**
```sql
-- Check ranking with bm25 scores
SELECT 
    rank,
    bm25(articles),
    title
FROM articles 
WHERE articles MATCH 'database'
ORDER BY rank;

-- Adjust BM25 parameters if needed
INSERT INTO articles_config(config, val) VALUES ('rank', 'bm25(articles, 1.5, 0.8, 0.4)');
```

**Slow searches:**
```sql
-- Optimize table
INSERT INTO articles(articles) VALUES('optimize');

-- Check table size
SELECT 
    page_count * page_size AS size_bytes
FROM pragma_page_count('articles'), pragma_page_size('articles');

-- Reindex if corrupted
INSERT INTO articles(articles) VALUES('reindex');
```

**Memory issues:**
```sql
-- Reduce page size for smaller memory footprint
INSERT INTO docs_config(config, val) VALUES ('page_size', '2048');

-- Increase merge limit for better compression
INSERT INTO docs_config(config, val) VALUES ('merge_limit', '8');
```

## Comparison: FTS3 vs FTS5

| Feature | FTS3 | FTS5 |
|---------|------|------|
| Performance | Good | Better |
| Disk usage | Higher | Lower |
| Prefix search | Limited | Full support |
| Language support | Basic | Advanced (unicode61) |
| Custom tokenizers | Yes | Yes (easier API) |
| Snippets | Basic | Advanced |
| Ranking | BM25 | Configurable BM25 |
| Contentless tables | No | Yes |
| Language IDs | No | Yes |

**Recommendation:** Use FTS5 for all new applications. Migrate existing FTS3 tables to FTS5 for better performance and features.
