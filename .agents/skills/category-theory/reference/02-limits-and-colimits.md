## Universal Properties

A **universal property** characterizes an object by its relationships to other objects, not by internal structure. This is the core method of category theory: define objects "up to unique isomorphism" via how morphisms factor through them.

The pattern: given a diagram of objects and morphisms, find an object *U* with morphisms to/from the diagram such that any other object with compatible morphisms factors uniquely through *U*.

**Example — Empty set**: In **Set**, the empty set is the unique set with no elements. Categorically, it is characterized as the terminal object in the category of sets over a fixed set (or equivalently, the initial object in **Set**).

## Products and Coproducts

**Product** of objects *A* and *B*: an object *A × B* with projections *π*1 : *A × B* → *A* and *π*2 : *A × B* → *B* such that for any object *X* with morphisms *f* : *X* → *A* and *g* : *X* → *B*, there exists a unique *⟨f, g⟩* : *X* → *A × B* with *π*1 ∘ *⟨f, g⟩* = *f* and *π*2 ∘ *⟨f, g⟩* = *g*.

**Coproduct** (dual): an object *A + B* with injections *ι*1 : *A* → *A + B* and *ι*2 : *B* → *A + B* such that for any *X* with *f* : *A* → *X* and *g* : *B* → *X*, there exists unique *[f, g]* : *A + B* → *X*.

**In Set:**
- Product = cartesian product (*A × B*), projections pick components
- Coproduct = disjoint union (*A ⊔ B*), injections embed each set

**In programming (Haskell):**
- Product types: `(a, b)` — tuples
- Coproduct types: `Either a b` — tagged unions
- Algebraic Data Types are built from products and coproducts

## Pullbacks and Pushouts

**Pullback** (fibered product): Given *f* : *A* → *C* and *g* : *B* → *C*, the pullback is an object *P* with morphisms *p*1 : *P* → *A* and *p*2 : *P* → *B* such that *f* ∘ *p*1 = *g* ∘ *p*2, universal among such squares.

**In Set**: The pullback of *f* : *A* → *C* and *g* : *B* → *C* is {(*a*, *b*) ∈ *A × B* | *f(a)* = *g(b)*}.

**Pushout** (dual): Given *f* : *C* → *A* and *g* : *C* → *B*, the pushout is an object *Q* with morphisms from *A* and *B* that agree on *C*, universal among such diagrams.

**In Set**: The pushout glues *A* and *B* along their common image of *C*.

## Equalizers and Coequalizers

**Equalizer** of *f*, *g* : *A* → *B*: an object *E* with *e* : *E* → *A* such that *f* ∘ *e* = *g* ∘ *e*, universal among such morphisms.

**In Set**: *E* = {*a* ∈ *A* | *f(a)* = *g(a)*}, and *e* is the inclusion.

**Kernel** (in additive categories): The equalizer of *f* and the zero morphism.

**Coequalizer** (dual): An object *Q* with *q* : *B* → *Q* such that *q* ∘ *f* = *q* ∘ *g*, universal.

**In Set**: *Q* is the quotient of *B* by the equivalence relation generated by *f(a)* ~ *g(a)* for all *a*.

## General Limits and Colimits

A **limit** of a diagram *D* : **J** → **C** (where **J** is an index category) is a cone from an object *L* to the diagram, universal among all cones. A **colimit** is the dual — a cocone from the diagram to an object *Colim D*.

**Special cases:**
- **Terminal object**: Limit of the empty diagram (unique morphism from every object)
- **Initial object**: Colimit of the empty diagram (unique morphism to every object)
- **Product**: Limit of a discrete diagram (no non-identity morphisms)
- **Coproduct**: Colimit of a discrete diagram
- **Inverse limit** (projective limit): Limit of a diagram indexed by a directed set going "backward"
- **Direct limit** (inductive limit/colimit): Colimit of a diagram indexed by a directed set

A category is **complete** if all small limits exist, **cocomplete** if all small colimits exist.

## Adjunctions

An **adjunction** is a pair of functors *F* : **C** → **D** and *G* : **D** → **C** such that there is a natural bijection:

Hom_D(*F(X)*, *Y*) ≅ Hom_C(*X*, *G(Y)*)

for all *X* in **C** and *Y* in **D**. We write *F* ⊣ *G* and say *F* is the **left adjoint**, *G* is the **right adjoint**.

**Unit and counit:**
- Unit *η* : Id_C ⇒ *G* ∘ *F* (from the identity on each *X* to *G(F(X)*)
- Counit *ε* : *F* ∘ *G* ⇒ Id_D

Satisfying triangle identities: (*ε* ∘ *F*) ∘ (*F* ∘ *η*) = Id_F and (*G* ∘ *ε*) ∘ (*η* ∘ *G*) = Id_G.

**Key examples:**
- **Free/Forgetful**: Free monoid ⊣ Forgetful (Mon → Set). The free monoid on a set *S* is the set of finite lists of elements of *S*.
- **Tensor-Hom adjunction**: In vector spaces, Hom(*V ⊗ W*, *U*) ≅ Hom(*V*, Hom(*W*, *U*))
- **Product/Exponential**: In cartesian closed categories, Hom(*A × B*, *C*) ≅ Hom(*A*, *C^B*)

Adjunctions are the abstract view of universal properties. Every universal property gives rise to an adjunction.

## The Yoneda Lemma

The **Yoneda embedding** maps an object *X* to the representable functor Hom(*X*, –) : **C** → **Set**. The **Yoneda lemma** states:

For any functor *F* : **C** → **Set** and object *X* in **C**:

Nat(Hom(*X*, –), *F*) ≅ *F(X)*

That is, natural transformations from the representable functor Hom(*X*, –) to *F* are in one-to-one correspondence with elements of *F(X)*.

**Corollary (Yoneda embedding is fully faithful)**: Nat(Hom(*X*, –), Hom(*Y*, –)) ≅ Hom(*Y*, *X*). Two objects are isomorphic iff their representable functors are naturally isomorphic.

The Yoneda lemma says an object is completely determined by its relationships to all other objects.

## Representable Functors

A functor *F* : **C**op → **Set** is **representable** if it is naturally isomorphic to Hom(–, *X*) for some object *X*. The object *X* is called a **representing object**.

**Example**: In **Vect**, the dual space functor *V* ↦ *V*\* = Hom(*V*, *k*) is representable (represented by the base field *k*).

A representable functor preserves all limits that exist in **C** (as a contravariant functor, it turns colimits into limits).
