# Object Methods

Object methods provide property access, merging, cloning, picking, omitting, and transformation utilities. Lodash handles deep paths, prototype chains, and custom merge strategies.

## Property Access

### `_.get(object, path, [defaultValue])`

Gets the value at path of object. Returns defaultValue if resolved value is undefined. Supports deep paths as array or dot-notation string.

```js
var object = { 'a': [{ 'b': { 'c': 3 } }] };
_.get(object, 'a[0].b.c');           // → 3
_.get(object, ['a', '0', 'b', 'c']); // → 3
_.get(object, 'a.b.c', 'default');   // → 'default'
```

### `_.has(object, path)` / `_.hasIn(object, path)`

Checks if path is a direct property (`has`) or exists in the prototype chain (`hasIn`). Returns false for undefined values even if the key exists.

```js
var object = { 'a': { 'b': 2 } };
var other = _.create({ 'a': _.create({ 'b': 2 }) });
_.has(object, 'a');     // → true
_.has(object, 'a.b');   // → true
_.has(object, 'a.b.c'); // → false
_.hasIn(other, 'a');    // → true (inherited)
```

### `_.result(object, path, [defaultValue])`

Like `get` but if the resolved value is a function, it's invoked with the object as this and its result is returned.

```js
_.result({ 'a': [{ 'b': function() { return 3; } }] }, 'a[0].b');
// → 3
```

## Setting

### `_.set(object, path, value)` / `_.setWith(object, path, value, [customizer])`

Sets the value at path. Creates undefined paths as needed. `setWith` uses customizer to generate objects for missing paths.

```js
_.set({}, 'a.b.c', 1);
// → { 'a': { 'b': { 'c': 1 } } }

var customizer = function(index, key) {
  if (_.isInteger(index)) {
    return [];
  }
};
_.setWith({}, '[0][1]', 'a', customizer);
// → [[, 'a']]
```

### `_.unset(object, path)`

Removes the property at path. Returns true if deleted.

```js
var object = { 'a': [{ 'b': { 'c': 7 } }] };
_.unset(object, 'a[0].b.c');
// → true
console.log(object); // → { 'a': [{ 'b': {} }] }
```

### `_.update(object, path, updater)` / `_.updateWith()`

Like set but the value is generated by invoking updater with the current value. Creates undefined paths as needed.

```js
_.update({ 'a': { 'b': 2 } }, 'a.b', function(n) { return n * 3; });
// → { 'a': { 'b': 6 } }
```

## Merging and Assigning

### `_.assign(object, [sources])`

Copies own enumerable string-keyed properties from sources to object. Processes left to right — later sources override earlier ones. Uses Object.assign semantics (shallow copy).

```js
function Foo() { this.a = 1; }
function Bar() { this.b = 2; }
Foo.prototype.c = 3;
Bar.prototype.d = 4;
_.assign({ 'a': 0 }, new Foo, new Bar);
// → { 'a': 1, 'b': 2 }
```

### `_.assignIn(object, [sources])` (alias: `_.extend`)

Like `assign` but also copies inherited enumerable properties.

### `_.assignWith()` / `_.assignInWith()`

Like assign/assignIn but with a customizer function `(objValue, srcValue, key, object, source)` to resolve conflicts. Return undefined to fall back to default behavior.

### `_.defaults(object, [sources])` / `_.defaultsDeep(object, [sources])`

Assigns properties from sources only if the property is undefined in object. `defaultsDeep` does this recursively for nested objects.

```js
_.defaults({ 'a': 1 }, { 'a': 3, 'b': 2 });
// → { 'a': 1, 'b': 2 }
```

### `_.merge(object, [sources])` / `_.mergeWith(object, sources, customizer)`

Recursively merges own and inherited enumerable properties. Arrays and objects are merged recursively, other values and properties of source objects overwrite object properties.

```js
var objects = [{ 'a': 1 }, { 'b': 2 }];
var deep = _.merge(objects, { 'a': 3 });
// → { 'a': 3, 'b': 2 }

_.merge({ 'x': 1 }, { 'x': _.merge({ 'a': 1 }, { 'a': 2 }) });
// → { 'x': { 'a': 2 } }
```

## Selecting Properties

### `_.pick(object, [paths])` / `_.pickBy(object, [predicate=_.identity])`

Creates an object composed of picked properties. `pick` takes explicit paths; `pickBy` uses a predicate `(value, key)`.

```js
var object = { 'a': 1, 'b': '2', 'c': 3 };
_.pick(object, ['a', 'c']);      // → { 'a': 1, 'c': 3 }
_.pickBy(object, _.isNumber);    // → { 'a': 1, 'c': 3 }
```

### `_.omit(object, [paths])` / `_.omitBy(object, [predicate=_.identity])`

The inverse of pick — creates an object excluding specified properties.

```js
var object = { 'a': 1, 'b': '2', 'c': 3 };
_.omit(object, ['a', 'c']);      // → { 'b': '2' }
_.omitBy(object, _.isNumber);    // → { 'b': '2' }
```

## Inspection

### `_.keys(object)` / `_.keysIn(object)`

Gets own enumerable string-keyed property names (`keys`) or including inherited (`keysIn`).

```js
_.keys({ 'a': 1, 'b': 2 }); // → ['a', 'b']
```

### `_.values(object)` / `_.valuesIn(object)`

Gets own enumerable string-keyed property values.

```js
_.values({ 'a': 1, 'b': 2 }); // → [1, 2]
```

### `_.toPairs(object)` / `_.toPairsIn(object)` (aliases: `entries`, `entriesIn`)

Creates an array of key-value pairs.

```js
_.toPairs({ 'a': 1, 'b': 2 }); // → [['a', 1], ['b', 2]]
```

### `_.invert(object)` / `_.invertBy(object, [iteratee=_.identity])`

Creates an object with values as keys and keys as values. `invertBy` groups by iteratee result.

```js
_.invert({ 'a': 1, 'b': 2 });     // → { '1': 'a', '2': 'b' }
_.invertBy({ 'a': 1, 'b': 2, 'c': 1 }, Number);
// → { '1': ['a', 'c'], '2': ['b'] }
```

### `_.functions(object)` / `_.functionsIn(object)`

Gets the method names (own or including inherited).

## Transformation

### `_.transform(object, [iteratee=_.identity], [accumulator])`

An alternative to `reduce` for objects. Iterates over properties and accumulates results. If accumulator not provided, an empty array is used if object is array-like, otherwise an empty object.

```js
_.transform({ 'a': 1, 'b': 2 }, function(result, n, key) {
  result[key] = n * 3;
});
// → { 'a': 3, 'b': 6 }

_.transform([1, 2], function(result, n) {
  result.push(n * 3);
}, []);
// → [3, 6]
```

## Object Creation

### `_.create(prototype, [properties])`

Creates a new object with prototype and optional properties.

```js
function Shape() { this.x = 0; }
function Circle() { Shape.call(this); }
Circle.prototype = _.create(Shape.prototype, {
  'constructor': Circle
});
var circle = new Circle;
circle instanceof Circle; // → true
circle instanceof Shape;  // → true
```

### `_.defaultsDeep()` with object creation

Common pattern for deep default configuration:

```js
var config = _.defaultsDeep(
  { 'server': { 'timeout': 5000 } },
  { 'server': { 'host': 'localhost', 'port': 3000 } }
);
// → { 'server': { 'timeout': 5000, 'host': 'localhost', 'port': 3000 } }
```

## Property Iteration

### `_.forIn(object, [iteratee])` / `_.forInRight()`

Iterates over own and inherited enumerable properties. Iteratee receives `(value, key, object)`.

### `_.forOwn(object, [iteratee])` / `_.forOwnRight()`

Iterates over own enumerable properties only.

```js
_.forOwn({ 'a': 1, 'b': 2 }, function(value, key) {
  console.log(key);
});
// → logs 'a', then 'b' (or 'b', then 'a')
```

### `_.at(object, [paths])`

Gets values at multiple paths simultaneously.

```js
var object = { 'a': 1, 'b': 2, 'c': 3 };
_.at(object, ['a', 'c']); // → [1, 3]
```
