# Internationalization (i18n)

Day.js has great support for internationalization with 80+ locales. None are included in your build unless you explicitly load them, keeping bundle size minimal.

## Loading Locales

**Node.js:**

```javascript
import 'dayjs/locale/es'        // load Spanish
// or
require('dayjs/locale/es')
```

**Browser (CDN):**

Locales are available as individual scripts:

```html
<script src="https://cdn.jsdelivr.net/npm/dayjs/locale/es.js"></script>
```

## Changing Locale Globally

Set the locale for all Day.js instances:

```javascript
import 'dayjs/locale/es'
dayjs.locale('es')              // use Spanish globally
```

## Changing Locale Locally

Override locale for a specific instance without affecting the global setting:

```javascript
import 'dayjs/locale/zh-cn'
dayjs('2018-05-05').locale('zh-cn').format()
// Uses Chinese Simplified for this instance only
```

Locale can be chained with other operations:

```javascript
dayjs().locale('fr').format('dddd, MMMM D')
```

## Checking Current Locale

```javascript
dayjs.locale()                  // current global locale (e.g., 'en')
dayjs().locale()                // current instance locale
```

## Locale Data (LocaleData plugin)

Access properties of the currently loaded locale:

```javascript
import localeData from 'dayjs/plugin/localeData'
dayjs.extend(localeData)

// Global locale data
const globalLD = dayjs.localeData()
globalLD.firstDayOfWeek()       // 0 (Sunday) or 1 (Monday), locale-dependent
globalLD.months()               // ['January', 'February', ...]
globalLD.monthsShort()          // ['Jan', 'Feb', ...]
globalLD.weekdays()             // ['Sunday', 'Monday', ...]
globalLD.weekdaysShort()        // ['Sun', 'Mon', ...]
globalLD.weekdaysMin()          // ['Su', 'Mo', ...]

// Instance locale data
const instanceLD = dayjs().localeData()
instanceLD.firstDayOfWeek()
```

Some methods accept a Day.js instance for context-sensitive values:

```javascript
globalLD.months(dayjs())        // stands (context-sensitive) month names
globalLD.weekdays(dayjs())      // stands (context-sensitive) weekday names
```

## Locale JSON File

Each release includes a `locale.json` file in the root with the list of all supported locales.

## Customizing Locales

### UpdateLocale Plugin

Update a locale's properties after loading:

```javascript
import updateLocale from 'dayjs/plugin/updateLocale'
dayjs.extend(updateLocale)

dayjs.updateLocale('en', {
  months: ['Janvier', 'Février', /* ... */],
  weekdays: ['Dimanche', 'Lundi', /* ... */]
})
```

### Relative Time Customization

Customize relative time strings in the locale configuration. Keys map to ranges:

- `s` — 0 to 44 seconds (`'a few seconds ago'`)
- `m` — 45 to 89 seconds (`'a minute ago'`)
- `mm` — 90 seconds to 44 minutes (`'%d minutes ago'`)
- `h` — 45 to 89 minutes (`'an hour ago'`)
- `hh` — 90 minutes to 21 hours (`'%d hours ago'`)
- `d` — 22 to 35 hours (`'a day ago'`)
- `dd` — 36 hours to 25 days (`'%d days ago'`)
- `M` — 26 to 45 days (`'a month ago'`)
- `MM` — 46 days to 10 months (`'%d months ago'`)
- `y` — 11 months to 17 months (`'a year ago'`)
- `yy` — 18+ months (`'%d years ago'`)

### Calendar Time Customization

Customize calendar output strings:

- `sameDay` — `'Today at h:mm A'`
- `nextDay` — `'Tomorrow at h:mm A'`
- `nextWeek` — `'dddd at h:mm A'`
- `lastDay` — `'Yesterday at h:mm A'`
- `lastWeek` — `'Last dddd at h:mm A'`
- `sameElse` — `'M/D/YYYY'`

### Adding a New Locale

You can contribute a locale by opening a pull request on the Day.js repository. Each locale is a self-contained JavaScript module that exports month names, weekday names, relative time strings, and calendar patterns.
