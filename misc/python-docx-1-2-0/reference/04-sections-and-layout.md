## Accessing Sections

```python
sections = document.sections
count = len(sections)
section = sections[0]

for section in sections:
    print(section.start_type)
```

Most documents have a single section. The `sections` collection supports indexing and iteration. Guard against empty collections when processing unpredictable files:

```python
if len(document.sections) == 0:
    # Handle edge case
    pass
```

## Adding Sections

```python
from docx.enum.section import WD_SECTION

new_section = document.add_section(WD_SECTION.NEW_PAGE)
# Options: NEW_PAGE, EVEN_PAGE, ODD_PAGE
```

Content added after `add_section()` appears in the new section. Default is `WD_SECTION.NEW_PAGE`.

## Page Dimensions and Orientation

### Current settings

```python
section = document.sections[0]
print(section.orientation)       # PORTRAIT (0) or LANDSCAPE (1)
print(section.page_width.inches)  # e.g. 8.5
print(section.page_height.inches) # e.g. 11.0
```

### Switching to landscape

```python
from docx.enum.section import WD_ORIENT

new_width, new_height = section.page_height, section.page_width
section.orientation = WD_ORIENT.LANDSCAPE
section.page_width = new_width
section.page_height = new_height
```

Swap width and height values before changing orientation.

## Margins

Seven properties control where text appears on the page:

```python
from docx.shared import Inches

section.left_margin = Inches(1.25)
section.right_margin = Inches(1.25)
section.top_margin = Inches(1.0)
section.bottom_margin = Inches(1.0)
section.gutter = Inches(0.25)          # Extra space for binding
section.header_distance = Inches(0.5)  # Distance from top edge to header
section.footer_distance = Inches(0.5)  # Distance from bottom edge to footer
```

All values are `Length` objects (EMU internally). Default document margins are typically 1.25" left/right and 1" top/bottom.

## Headers and Footers

Headers and footers are linked to sections, allowing different headers per section.

### Accessing header/footer

```python
section = document.sections[0]
header = section.header
footer = section.footer
```

A `_Header`/`_Footer` object always exists, even when no header is defined. Check `is_linked_to_previous`:

```python
if header.is_linked_to_previous:
    # No header definition — inherits from previous section
    pass
```

### Adding a simple header

```python
paragraph = header.paragraphs[0]
paragraph.text = 'Title of my document'
```

Adding content automatically creates a header definition and sets `is_linked_to_previous` to `False`.

### Zoned headers (left/center/right)

Use tab characters with the built-in `Header` style:

```python
paragraph = header.paragraphs[0]
paragraph.text = 'Left Text\tCenter Text\tRight Text'
paragraph.style = document.styles['Header']
```

The `Header` style includes pre-configured tab stops for left, center, and right alignment.

### Removing a header

```python
header.is_linked_to_previous = True
```

Content is irreversibly deleted. The section inherits from the previous section's header.

### Creating an independent header definition

```python
header.is_linked_to_previous = False
```

Creates a new header definition with one empty paragraph, breaking inheritance from the previous section.

### Multi-section header behavior

1. Each section can have its own header definition (but doesn't have to)
2. A section without a definition inherits from the first prior section that has one
3. `is_linked_to_previous = True` means no local definition (inheritance)
4. Editing an inherited header actually edits the source section's header
5. To create independent content, first set `is_linked_to_previous = False`

### Footer

Footers work identically to headers — same properties and methods:

```python
footer.paragraphs[0].text = 'Page {PAGE}'
```

Note: Page number fields (`{PAGE}`) are not automatically generated by python-docx. They require inserting OOXML field codes manually.
