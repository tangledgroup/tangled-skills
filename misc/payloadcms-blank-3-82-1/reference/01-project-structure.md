
## Route Groups

Next.js route groups in parentheses `(group-name)` organize files without affecting URL paths. The blank template uses two groups:

- **`(payload)/`** — Contains the admin panel. Payload serves its React UI at this route group. The layout wraps content with `RootLayout` from `@payloadcms/next/layouts`, which provides server function handling and admin import maps.

- **`(frontend)/`** — Contains public-facing pages. This is where you build your website or application frontend. The home page demonstrates using Payload's Local API to check authentication and display dynamic content.

## Source Directory Conventions

All application code lives under `src/`. Key conventions:

- `payload.config.ts` is the single entry point for all Payload configuration — collections, plugins, database adapter, editor, and admin settings are defined here
- `collections/` contains one file per collection, each exporting a `CollectionConfig` object
- `payload-types.ts` is auto-generated by `payload generate:types` — never edit it manually
- `app/` follows Next.js App Router conventions with server components

## Frontend Page Pattern

The home page at `app/(frontend)/page.tsx` demonstrates the standard pattern for accessing Payload data in server components:

```tsx
import { headers as getHeaders } from 'next/headers.js'
import { getPayload } from 'payload'
import config from '@/payload.config'

export default async function HomePage() {
  const headers = await getHeaders()
  const payloadConfig = await config
  const payload = await getPayload({ config: payloadConfig })
  const { user } = await payload.auth({ headers })

  // Use payload.find(), payload.findByID(), etc.
  return <div>Welcome</div>
}
```

This pattern — importing the config, calling `getPayload()`, then using the Local API — is the recommended approach for server-side data access in Payload 3.x with Next.js App Router.

## Admin Layout Pattern

The admin layout at `app/(payload)/layout.tsx` uses Payload's `RootLayout` component with a server function client:

```tsx
import config from '@payload-config'
import { handleServerFunctions, RootLayout } from '@payloadcms/next/layouts'
import type { ServerFunctionClient } from 'payload'

const serverFunction: ServerFunctionClient = async function (args) {
  'use server'
  return handleServerFunctions({
    ...args,
    config,
    importMap,
  })
}

const Layout = ({ children }: { children: React.ReactNode }) => (
  <RootLayout config={config} importMap={importMap} serverFunction={serverFunction}>
    {children}
  </RootLayout>
)
```

This setup enables Payload's server function system, which allows the admin UI to call server-side operations (like saving documents) from React components.
