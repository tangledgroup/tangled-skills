# Tracing

## TracerProvider

The entry point of the Tracing API. Provides access to Tracers.

- SHOULD provide a way to set/register and access a global default TracerProvider
- MUST allow creating an arbitrary number of TracerProvider instances
- Configuration (SpanProcessors, IdGenerator, SpanLimits, Sampler) is owned by the TracerProvider
- If configuration is updated, it MUST apply to all already returned Tracers

### Get a Tracer

```
tracer = tracer_provider.get_tracer(name, version, schema_url, attributes)
```

- `name` (required): Uniquely identifies the instrumentation scope
- `version` (optional): Library version string
- `schema_url` (optional): Schema URL for emitted telemetry
- `attributes` (optional): Instrumentation scope attributes

If an invalid name is specified, a working Tracer MUST be returned as fallback (name set to empty string).

### Shutdown / ForceFlush

- **Shutdown**: Cleanup. MUST be called only once. After shutdown, subsequent `get_tracer` calls SHOULD return no-op Tracers.
- **ForceFlush**: Immediately export all unexported spans. MAY have a configurable timeout.

## Tracer `enabled` API (Stable in 1.56.0)

To help users avoid performing computationally expensive operations when creating `Span`s, a `Tracer` SHOULD provide this `Enabled` API.

There are currently no required parameters for this API. Parameters can be added in the future, therefore, the API MUST be structured in a way for parameters to be added.

This API MUST return a language idiomatic boolean type. A returned value of `true` means the `Tracer` is enabled for the provided arguments, and a returned value of `false` means the `Tracer` is disabled for the provided arguments.

The returned value is not always static — it can change over time. The API SHOULD be documented that instrumentation authors need to call this API each time they create a new `Span` to ensure they have the most current value.

```python
if tracer.enabled():
    with tracer.start_as_current_span("expensive-operation") as span:
        # Only perform expensive work when tracing is enabled
        result = compute_heavy_task()
        span.set_attribute("result.size", len(result))
```

## Span Creation and Lifetime

A span represents an operation within a transaction. Created through a Tracer:

```python
with tracer.start_as_current_span("operation-name") as span:
    span.set_attribute("http.method", "GET")
    span.add_event("processing.complete", {"duration_ms": 42})
```

### Span Fields

- **Operation name**: String identifying the operation
- **Start and end timestamps**: Nanosecond precision (minimal: milliseconds)
- **Attributes**: Key-value pairs (key is string, value is AnyValue)
- **Events**: Named timestamps with optional attributes
- **Status**: Ok, Error, or Unset
- **Links**: References to causally-related Spans (via SpanContext)
- **SpanKind**: Internal, Server, Client, Producer, Consumer

### Determining Parent Span

When creating a span, the parent is determined from the Context:

1. If the Context contains a remote SpanContext → use it as parent
2. If the Context contains a local Span → use it as parent
3. Otherwise → root span (no parent)

## SpanContext

The propagated portion of a Span. Contains:

- **TraceId**: 16 randomly generated bytes, worldwide unique with practical sufficiency
- **SpanId**: 8 randomly generated bytes, globally unique
- **TraceFlags**: 1-byte bitmap. Sampling bit = mask `0x1`
- **TraceState**: Vendor-specific key-value pairs for additional tracing context

### IsValid / IsRemote

- `IsValid`: Returns true if TraceId and SpanId are not all zeros
- `IsRemote`: Returns true if the SpanContext was extracted from wire format (not locally created)

## Sampling

The Sampler decides whether a new span should be recorded and exported.

### Built-in Samplers

- **AlwaysOn**: Sample every trace
- **AlwaysOff**: Never sample
- **TraceIdRatioBased**: Samples based on a ratio derived from TraceId. Uses first byte of TraceId: `int(trace_id[0] & 0xFF) / 256.0 < probability`
- **ProbabilitySampler**: Configurable probability sampler with explicit randomness in TraceState
- **ParentBased**: Delegates to root sampler when no parent, otherwise follows parent's sampled flag
- **AlwaysRecord** (Stable in 1.56.0): Sampler decorator that ensures every span is passed to the `SpanProcessor`, even those that would normally be dropped. Converts `DROP` decisions from the wrapped sampler into `RECORD_ONLY` decisions, allowing processors to see all spans without sending them to exporters. Typically used for accurate span-to-metrics processing.
- **CompositeSampler** (Development): Combines multiple samplers
- **ComposableSampler** (Development): Interface for building custom composable samplers

### AlwaysRecord Sampler Details (Stable in 1.56.0)

`AlwaysRecord` wraps another sampler and modifies its behavior:

- Based on the decision from the wrapped root sampler, `AlwaysRecord` MUST behave as follows:
  - If wrapped sampler returns `RECORD_AND_SAMPLE` → return `RECORD_AND_SAMPLE`
  - If wrapped sampler returns `RECORD_ONLY` → return `RECORD_ONLY`
  - If wrapped sampler returns `DROP` → return `RECORD_ONLY` (not `DROP`)

This enables processors (e.g., span-to-metrics) to process every span while exporters skip the non-recorded ones.

### Sampling Requirements

- Root samplers set explicit randomness for non-random TraceIDs
- TraceID randomness MUST be preserved — do not overwrite explicit randomness
- The `ShouldSample` method receives parent context, trace ID, name, kind, attributes, and links

## Span Processors

Processors receive lifecycle events from spans.

### OnStart

Called when a span is started. The span is not yet recorded.

### OnEnd

Called when a span ends. The span is ready to be exported.

### Built-in Processors

- **Simple Processor**: Exports each span immediately on `OnEnd`
- **Batching Processor**: Batches spans for efficient export. Configurable with max queue size, schedule delay, export timeout, and export max batch size

## Span Exporter

Interface for exporting spans:

- **Export(batch)**: Exports a batch of spans
- **Shutdown()**: Cleanup
- **ForceFlush()**: Flush pending exports

### Zipkin Exporter (Deprecated)

Zipkin exporter support is deprecated as of 1.55.0 and will be removed from the specification in December 2026. Existing stable Zipkin exporters MUST continue to be supported for at least one year after deprecation. Users may use the Zipkin exporter Collector component or `zipkin-otel` Zipkin server module instead.

## Trace IDs and Randomness

- TraceIDs should have sufficient randomness (128 bits)
- Implementations SHOULD support random TraceID generation
- ID generators (IdGenerator) can be customized per TracerProvider

## Span Limits

Configurable limits to control memory usage:

- Maximum attributes per span
- Maximum events per span
- Maximum links per span
- Maximum attribute length
