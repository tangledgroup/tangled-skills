# FTS5 Full-Text Search

## Overview

FTS5 is an SQLite virtual table module that provides full-text search functionality. It indexes text content and supports efficient term lookup, phrase matching, prefix queries, proximity searches, and relevance ranking.

FTS5 is the successor to FTS3/FTS4 and is recommended for new projects. It is not enabled by default — compile with `-DSQLITE_ENABLE_FTS5=1`.

## Creating an FTS5 Table

```sql
CREATE VIRTUAL TABLE emails USING fts5(
    subject,
    body,
    content='source_table',
    content_rowid='source_id',
    tokenize='unicode61 language=en'
);
```

FTS5 tables cannot have type declarations, constraints, or PRIMARY KEY declarations in the CREATE statement. They have an implicit `rowid` column.

## Populating FTS5

Insert, update, and delete work like normal tables:

```sql
INSERT INTO emails (rowid, subject, body)
SELECT id, subject, body FROM messages;

UPDATE emails SET body = 'new content' WHERE rowid = 1;
DELETE FROM emails WHERE rowid = 1;
```

For external content tables, populate the source table and FTS5 will index automatically.

## Querying FTS5

Three ways to search:

```sql
-- MATCH operator (most common)
SELECT * FROM emails WHERE emails MATCH 'hello world';

-- Equals operator
SELECT * FROM emails WHERE emails = 'hello world';

-- Table-valued function syntax
SELECT * FROM emails('hello world');
```

## Query Syntax

**Term queries:**
```sql
SELECT * FROM docs WHERE docs MATCH 'sqlite';
```

**Phrase queries (exact order, adjacent tokens):**
```sql
SELECT * FROM docs WHERE docs MATCH '"full text search"';
```

**Prefix queries (terms starting with a prefix):**
```sql
SELECT * FROM docs WHERE docs MATCH 'sql*';
```

**Boolean operators:**
```sql
-- AND (default — all terms must match)
SELECT * FROM docs WHERE docs MATCH 'sqlite database';

-- OR
SELECT * FROM docs WHERE docs MATCH 'sqlite OR postgresql';

-- NOT (exclude documents containing a term)
SELECT * FROM docs WHERE docs MATCH 'sqlite NOT trigger';
```

**NEAR queries (terms within N tokens of each other):**
```sql
SELECT * FROM docs WHERE docs MATCH 'NEAR("full text", 5)';
```

**Column filters (search only specific columns):**
```sql
SELECT * FROM emails WHERE emails MATCH 'subject:hello';
SELECT * FROM emails WHERE emails MATCH 'subject:hello AND body:world';
```

## Tokenizers

Tokenizers determine how text is broken into searchable terms:

**unicode61** (default):
- Handles Unicode normalization
- Supports language-specific stemming
- Options: `language=xx` for stemming, `remove_duplicates=1`
```sql
tokenize='unicode61 language=en remove_duplicates=1'
```

**ascii**:
- Simple ASCII tokenizer, no Unicode handling
- Lowercases and splits on non-alphanumeric characters

**porter**:
- Like unicode61 but always applies English Porter stemming

**trigram**:
- Splits text into overlapping 3-character sequences
- Useful for fuzzy matching and non-tokenizable languages

## Prefix Indexes

Speed up prefix queries by storing additional indexes:

```sql
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    prefix='3,5,10',
    tokenize='unicode61'
);
```

This creates prefix indexes for 3, 5, and 10 character prefixes.

## Auxiliary Functions

FTS5 provides built-in functions usable in SELECT:

**bm25()** — Relevance ranking using the BM25 algorithm:
```sql
SELECT *, bm25(docs) AS rank
FROM docs WHERE docs MATCH 'sqlite tutorial'
ORDER BY rank;
```

**highlight()** — Wrap matching terms in markers:
```sql
SELECT highlight(docs, 0, '<b>', '</b>') FROM docs
WHERE docs MATCH 'sqlite';
```

**snippet()** — Extract a snippet of text around the match:
```sql
SELECT snippet(docs, 0, '...', '...', 30) FROM docs
WHERE docs MATCH 'database optimization';
```

## Special Commands

FTS5 tables accept special INSERT commands for maintenance:

```sql
-- Merge segment files (improves query performance)
INSERT INTO docs(docs) VALUES('merge');

-- Optimize the index
INSERT INTO docs(docs) VALUES('optimize');

-- Rebuild the index from external content
INSERT INTO docs(docs) VALUES('rebuild');

-- Integrity check
INSERT INTO docs(docs) VALUES('integrity-check');

-- Delete all content
INSERT INTO docs(docs) VALUES('delete-all');
```

Configuration options:
```sql
-- Set automerge threshold
INSERT INTO docs(docs) VALUES('automerge', 100);

-- Set page size
INSERT INTO docs(docs) VALUES('pgsz', 32768);
```

## External Content Tables

FTS5 can index content stored in a separate table:

```sql
CREATE VIRTUAL TABLE docs_fts USING fts5(
    title, body,
    content='documents',
    content_rowid='id'
);
```

Changes to the source table must trigger FTS5 updates via triggers or application logic. Contentless tables (`content=''`) store only the index without document text — useful when you only need document IDs.

## Comparison with FTS3/FTS4

Key differences:
- FTS5 uses a different, more compact index format
- FTS5 does not support the `snippet()` function's same parameters as FTS3
- FTS5 has the trigram tokenizer (not in FTS3)
- FTS5 auxiliary functions use different calling conventions
- Migration requires rebuilding the index
